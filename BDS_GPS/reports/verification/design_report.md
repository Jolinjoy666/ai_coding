# BDS_GPS 设计报告

## 1. 设计概述

### 1.1 设计目标
设计一个BDS/GPS卫星导航系统C/A码捕获与跟踪的Verilog模块，能够：
1. 从采样文件中捕获并跟踪GPS和BDS卫星的C/A码
2. 识别卫星的sv_num（卫星号）
3. 解调出每颗卫星的5位循环码

### 1.2 设计要求
- 支持两颗GPS卫星和两颗BDS卫星
- 系统时钟频率：16.368MHz
- GPS C/A码片速率：1.023MHz（16倍系统时钟）
- BDS C/A码片速率：2.046MHz（8倍系统时钟）
- 捕获算法：相关峰检测
- 跟踪算法：三路C/A码（e、p、l）
- 输出：卫星号(sv_num)和5位循环码

## 2. 解题思路

### 2.1 需求分析
1. **C/A码生成**：需要实现GPS和BDS的C/A码发生器
2. **捕获算法**：使用相关峰检测方法，移动C/A码初始相位，计算相关值
3. **跟踪算法**：使用三路C/A码（超前、基准、滞后），动态调整相位
4. **数据处理**：处理采样文件中的多颗卫星C/A码叠加数据

### 2.2 架构选择
选择**混合架构（时分复用）**：
- 共享相关器和累加器，节省FPGA资源
- 通过时间片调度处理多颗卫星
- 在资源使用和性能之间取得良好平衡

### 2.3 关键设计点
1. **时间片调度**：1ms时间片，顺序处理4颗卫星
2. **C/A码生成**：实现GPS和BDS的C/A码发生器
3. **相关计算**：计算C/A码与采样数据的相关值
4. **捕获检测**：比较相关值与阈值，判断捕获状态
5. **跟踪控制**：根据e、p、l三路相关值调整相位

## 3. Verilog源码和说明

### 3.1 顶层模块：bds_gps_top
**文件**：`rtl/generated/bds_gps_top.sv`

**功能**：
- 管理时间片调度
- 协调各子模块工作
- 提供顶层接口

**关键信号**：
- `clk`：16.368MHz系统时钟
- `rst_n`：异步复位，低有效
- `sample_data[7:0]`：采样数据输入
- `satellite_num[5:0]`：卫星号输出
- `cycle_code[4:0]`：5位循环码输出

### 3.2 时间片调度器：time_slice_scheduler
**文件**：`rtl/generated/time_slice_scheduler.sv`

**功能**：
- 生成时间片控制信号
- 管理卫星处理顺序（GPS1 → GPS2 → BDS1 → BDS2）
- 控制时间片切换

**关键参数**：
- `TIME_SLICE_COUNTS`：16368（1ms时间片）
- `SATELLITE_NUM`：4（支持4颗卫星）

### 3.3 C/A码生成器：ca_code_generator
**文件**：`rtl/generated/ca_code_generator.sv`

**功能**：
- 生成GPS C/A码（G1和G2发生器）
- 生成BDS C/A码（G1和G2发生器）
- 根据卫星号选择G2输出
- 生成e、p、l三路C/A码

**关键设计**：
- GPS：10位移位寄存器，码周期1023
- BDS：11位移位寄存器，码周期2046
- 相位延迟：GPS超前/滞后4个时钟周期，BDS超前/滞后2个时钟周期

### 3.4 相关器：correlator
**文件**：`rtl/generated/correlator.sv`

**功能**：
- 计算C/A码与采样数据的相关值
- 支持三路C/A码（e、p、l）
- 实时计算相关值

**关键算法**：
- 符号转换：C/A码1→-1，0→+1
- 相关计算：采样数据 × C/A码符号
- 累加：1ms周期内累加相关值

### 3.5 累加器：accumulator
**文件**：`rtl/generated/accumulator.sv`

**功能**：
- 累加相关值（1ms周期）
- 提供累加结果
- 支持复位和清零

### 3.6 捕获检测器：acquisition_detector
**文件**：`rtl/generated/acquisition_detector.sv`

**功能**：
- 检测捕获状态
- 比较相关值与阈值
- 生成捕获完成信号

**关键参数**：
- `ACQUISITION_THRESHOLD`：捕获阈值（示例值1000）

### 3.7 跟踪控制器：tracking_controller
**文件**：`rtl/generated/tracking_controller.sv`

**功能**：
- 控制跟踪过程
- 调整C/A码相位
- 保持跟踪锁定状态

**状态机**：
- `TRACK_IDLE`：空闲状态
- `TRACK_LOCKED`：锁定状态
- `TRACK_ADJUST`：调整状态

### 3.8 输出接口：output_interface
**文件**：`rtl/generated/output_interface.sv`

**功能**：
- 输出卫星号和5位循环码
- 生成状态指示信号
- 管理输出时序

**卫星号映射**：
- 时间片00：GPS卫星1（sv_num=1）
- 时间片01：GPS卫星2（sv_num=2）
- 时间片10：BDS卫星1（sv_num=31）
- 时间片11：BDS卫星2（sv_num=32）

## 4. 中间结果波形截图说明

### 4.1 时钟和复位波形
**说明**：
- 系统时钟频率：16.368MHz
- 异步复位，低有效
- 复位时所有寄存器初始化

**波形特征**：
- 时钟周期：61.08ns
- 复位释放后，系统开始工作

### 4.2 时间片调度波形
**说明**：
- 1ms时间片，顺序处理4颗卫星
- 时间片ID：00（GPS1）→ 01（GPS2）→ 10（BDS1）→ 11（BDS2）

**波形特征**：
- `time_slice_start`信号在每个时间片开始时脉冲
- `time_slice_end`信号在每个时间片结束时脉冲
- `time_slice_id`按顺序变化

### 4.3 C/A码生成波形
**说明**：
- GPS C/A码：10位移位寄存器，码周期1023
- BDS C/A码：11位移位寄存器，码周期2046

**波形特征**：
- GPS C/A码发生器输出伪随机序列
- BDS C/A码发生器输出伪随机序列
- e、p、l三路C/A码有相位差

### 4.4 相关值计算波形
**说明**：
- 相关器计算C/A码与采样数据的相关值
- 累加器累加1ms周期内的相关值

**波形特征**：
- 相关值在采样数据有效时累加
- 累加结果在时间片结束时更新

### 4.5 捕获检测波形
**说明**：
- 捕获检测器比较累加结果与阈值
- 当累加结果超过阈值时，捕获完成

**波形特征**：
- `threshold_exceeded`信号在累加结果超过阈值时变高
- `acquired`信号在时间片结束时更新

### 4.6 跟踪控制波形
**说明**：
- 跟踪控制器根据e、p、l三路相关值调整相位
- 保持p路相关值最大

**波形特征**：
- `tracking_state`信号在捕获后变高
- 根据相关值大小关系调整相位

### 4.7 输出接口波形
**说明**：
- 输出卫星号和5位循环码
- 状态指示信号

**波形特征**：
- `satellite_num`根据时间片ID变化
- `cycle_code`根据相关值正负关系解调
- `acquisition_done`和`tracking_locked`信号指示状态
