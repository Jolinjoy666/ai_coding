# BDS_GPS 选定架构

## 选择的架构：混合架构（时分复用）

## 选择理由

1. **资源效率**：共享相关器和累加器，节省FPGA资源
2. **性能平衡**：在资源使用和性能之间取得良好平衡
3. **可扩展性**：可以支持更多卫星，只需调整时间片
4. **实现复杂度**：比并行架构简单，比顺序架构复杂度适中
5. **验证友好**：验证复杂度适中，测试用例数量可控

## 架构概述

### 主要模块
1. **顶层控制器**：管理时间片调度和状态机
2. **C/A码生成器**：生成GPS和BDS的C/A码
3. **相关器**：计算C/A码与采样数据的相关值
4. **累加器**：累加相关值
5. **捕获检测器**：检测捕获状态
6. **跟踪控制器**：控制跟踪过程
7. **输出接口**：输出卫星号和5位循环码

### 数据流
```
采样数据输入 → 时间片分配 → 共享处理单元 → 结果存储 → 下一个时间片
```

### 时间片调度
- 时间片长度：1ms（一个C/A码周期）
- 卫星处理顺序：GPS1 → GPS2 → BDS1 → BDS2 → 循环
- 每个时间片内处理一颗卫星

### 关键设计点
1. **共享资源**：相关器和累加器在时间片内共享
2. **独立存储**：每颗卫星有独立的结果存储
3. **状态保持**：跟踪状态在时间片切换时保持
4. **时序控制**：严格的时间片切换时序

## 接口定义

### 输入接口
- `clk`：16.368MHz系统时钟
- `rst_n`：异步复位，低有效
- `sample_data[7:0]`：采样数据输入
- `sample_valid`：采样数据有效

### 输出接口
- `satellite_num[5:0]`：卫星号（sv_num）
- `cycle_code[4:0]`：5位循环码
- `acquisition_done`：捕获完成指示
- `tracking_locked`：跟踪锁定指示

## 资源估算

### 逻辑资源
- 寄存器：约500个
- LUT：约1000个
- 乘法器：约10个
- 存储器：约10KB

### 时钟频率
- 目标频率：16.368MHz
- 时序裕量：>10%

## 风险和缓解措施

### 风险1：时间片冲突
- **缓解**：合理安排时间片，避免冲突

### 风险2：资源争用
- **缓解**：为关键资源预留时间片

### 风险3：时序问题
- **缓解**：优化时间片切换逻辑，确保时序满足
