# BDS_GPS 微架构设计文档

## 1. 选定架构和需求重新陈述

### 1.1 选定架构
混合架构（时分复用）：共享相关器和累加器，通过时间片调度处理多颗卫星。

### 1.2 关键需求
- 支持两颗GPS卫星和两颗BDS卫星
- 系统时钟频率：16.368MHz
- GPS C/A码片速率：1.023MHz（16倍系统时钟）
- BDS C/A码片速率：2.046MHz（8倍系统时钟）
- 捕获算法：相关峰检测
- 跟踪算法：三路C/A码（e、p、l）
- 输出：卫星号(sv_num)和5位循环码

## 2. 模块分解和职责

### 2.1 顶层模块：bds_gps_top
**职责**：
- 管理时间片调度
- 协调各子模块工作
- 提供顶层接口

### 2.2 时间片调度器：time_slice_scheduler
**职责**：
- 生成时间片控制信号
- 管理卫星处理顺序
- 控制时间片切换

### 2.3 C/A码生成器：ca_code_generator
**职责**：
- 生成GPS C/A码（G1和G2发生器）
- 生成BDS C/A码（G1和G2发生器）
- 根据卫星号选择G2输出

### 2.4 相关器：correlator
**职责**：
- 计算C/A码与采样数据的相关值
- 支持三路C/A码（e、p、l）
- 实时计算相关值

### 2.5 累加器：accumulator
**职责**：
- 累加相关值（1ms周期）
- 提供累加结果
- 支持复位和清零

### 2.6 捕获检测器：acquisition_detector
**职责**：
- 检测捕获状态
- 比较相关值与阈值
- 生成捕获完成信号

### 2.7 跟踪控制器：tracking_controller
**职责**：
- 控制跟踪过程
- 调整C/A码相位
- 保持跟踪锁定状态

### 2.8 输出接口：output_interface
**职责**：
- 输出卫星号和5位循环码
- 生成状态指示信号
- 管理输出时序

## 3. 接口、协议、时钟/复位域和延迟

### 3.1 顶层接口
```systemverilog
module bds_gps_top (
    // 时钟和复位
    input  logic        clk,           // 16.368MHz系统时钟
    input  logic        rst_n,         // 异步复位，低有效
    
    // 采样数据输入
    input  logic [7:0]  sample_data,   // 采样数据
    input  logic        sample_valid,  // 采样数据有效
    
    // 输出接口
    output logic [5:0]  satellite_num, // 卫星号（sv_num）
    output logic [4:0]  cycle_code,    // 5位循环码
    output logic        acquisition_done, // 捕获完成指示
    output logic        tracking_locked   // 跟踪锁定指示
);
```

### 3.2 内部接口
- 时间片控制信号：`time_slice_id[1:0]`（00: GPS1, 01: GPS2, 10: BDS1, 11: BDS2）
- C/A码输出：`ca_code_e`, `ca_code_p`, `ca_code_l`（超前、基准、滞后）
- 相关值输出：`corr_value_e`, `corr_value_p`, `corr_value_l`
- 累加结果：`accum_result`
- 捕获状态：`acquired`, `threshold_exceeded`

### 3.3 时钟/复位域
- 单一时钟域：16.368MHz
- 异步复位，同步释放
- 复位时所有寄存器初始化

### 3.4 延迟要求
- 捕获延迟：尽量快速（顺序处理4颗卫星）
- 跟踪延迟：实时调整（每个时间片内）

## 4. 参数和合法范围

### 4.1 系统参数
- `CLK_FREQ`：16.368MHz
- `GPS_CA_RATE`：1.023MHz
- `BDS_CA_RATE`：2.046MHz
- `GPS_CA_CHIP_WIDTH`：16（系统时钟周期）
- `BDS_CA_CHIP_WIDTH`：8（系统时钟周期）

### 4.2 C/A码参数
- `GPS_REG_WIDTH`：10
- `BDS_REG_WIDTH`：11
- `GPS_CA_PERIOD`：1023
- `BDS_CA_PERIOD`：2046

### 4.3 时间片参数
- `TIME_SLICE_MS`：1（1ms时间片）
- `SATELLITE_NUM`：4（支持4颗卫星）

### 4.4 阈值参数
- `ACQUISITION_THRESHOLD`：捕获阈值（待定）
- `TRACKING_THRESHOLD`：跟踪阈值（待定）

## 5. FSM、计数器、仲裁、错误处理、刷新/清零和超时行为

### 5.1 主状态机（时间片调度器）
```
IDLE → SLICE_START → PROCESSING → SLICE_END → IDLE
```

### 5.2 捕获状态机
```
ACQ_IDLE → ACQ_SEARCH → ACQ_FOUND → ACQ_DONE
```

### 5.3 跟踪状态机
```
TRACK_IDLE → TRACK_LOCKED → TRACK_ADJUST → TRACK_LOCKED
```

### 5.4 计数器
- 时间片计数器：计数1ms时间片
- C/A码相位计数器：计数C/A码相位
- 系统时钟计数器：计数系统时钟周期

### 5.5 仲裁
- 时间片调度器仲裁卫星处理顺序
- 优先级：固定顺序（GPS1 → GPS2 → BDS1 → BDS2）

### 5.6 错误处理
- 捕获失败：继续搜索下一个相位
- 跟踪失锁：重新进入捕获状态
- 时间片超时：强制切换到下一个时间片

### 5.7 刷新/清零
- 时间片切换时清零相关器和累加器
- 复位时清零所有寄存器

### 5.8 超时行为
- 时间片超时：1ms时间片结束
- 捕获超时：可选，防止无限搜索

## 6. 数据通路、流水线、存储器/寄存器行为和冲突语义

### 6.1 数据通路
```
采样数据输入 → 时间片选择 → C/A码生成 → 相关计算 → 累加 → 阈值比较 → 状态更新
```

### 6.2 流水线
- 无流水线，顺序处理
- 每个时间片内顺序执行：C/A码生成 → 相关计算 → 累加 → 阈值比较

### 6.3 存储器/寄存器行为
- 采样数据存储：FIFO或RAM（存储1ms数据）
- C/A码存储：查找表或实时生成
- 相关值存储：寄存器
- 累加结果存储：寄存器
- 跟踪状态存储：寄存器

### 6.4 冲突语义
- 时间片切换时，确保所有操作完成
- 共享资源（相关器、累加器）在时间片内独占使用
- 无并发访问冲突

## 7. 复位、CDC/RDC、断言和验证意图

### 7.1 复位策略
- 异步复位，同步释放
- 复位时所有寄存器初始化为特定值
- GPS C/A码发生器：全1
- BDS C/A码发生器：01010101010

### 7.2 CDC/RDC
- 单一时钟域，无CDC问题
- 异步复位，需要同步释放电路

### 7.3 断言
- 时间片切换时序断言
- C/A码周期正确性断言
- 相关值范围断言
- 状态机转换断言

### 7.4 验证意图
- 功能验证：捕获和跟踪功能正确
- 时序验证：满足时序要求
- 资源验证：资源使用符合预期
- 边界验证：边界条件处理正确

## 8. 风险和开放问题

### 8.1 风险
1. **时间片冲突**：时间片切换可能导致状态丢失
2. **资源争用**：共享资源可能导致性能瓶颈
3. **时序问题**：时间片切换时序可能不满足
4. **捕获效率**：顺序处理可能导致捕获时间较长

### 8.2 开放问题
1. 捕获阈值的具体设定值
2. 跟踪算法的详细参数
3. 5位循环码的具体编码方式
4. 采样文件的具体格式
