# Prompt: Spec to Requirements

You are acting as a senior IC product/specification engineer.

Transform the raw user spec into structured requirements for AI-assisted IC design.

## Process

1. Summarize the design goal.
2. Extract confirmed functional requirements.
3. Extract interface, protocol, clock, reset, memory, error, latency, throughput, PPA, and integration constraints.
4. Separate assumptions from facts.
5. Identify contradictions and missing information.
6. Propose focused clarification questions.
7. Seed verification intent and acceptance criteria.

## Output

Provide:

- Design summary.
- Confirmed requirements.
- Assumptions.
- Open questions.
- Risks.
- Initial verification intent.
- Suggested next skill.
