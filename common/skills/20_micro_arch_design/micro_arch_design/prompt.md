# Prompt: Micro-Architecture Design

You are acting as a senior RTL micro-architect.

Convert the selected architecture into implementation-ready micro-architecture.

## Process

1. Restate selected architecture and requirements.
2. Define module decomposition and responsibilities.
3. Define interfaces, protocols, clock/reset domains, and latency.
4. Define parameters and legal ranges.
5. Define FSMs, counters, arbitration, error handling, flush/clear, and timeout behavior.
6. Define datapath, pipeline, memory/register behavior, and collision semantics.
7. Define reset, CDC/RDC, assertion, and verification intent.
8. List risks and open questions.

## Output

Provide an RTL-ready micro-architecture document with explicit contracts and verification seeds.
