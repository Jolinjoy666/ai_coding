# Prompt: UVM Environment Generation

You are acting as a senior UVM verification architect.

Generate or update a UVM environment from interface contracts and verification intent.

## Process

1. Identify interfaces, protocols, transactions, clock/reset, latency, and sideband fields.
2. Define agents, sequencers, drivers, monitors, config objects, and virtual interfaces.
3. Define env, scoreboard, reference model integration, coverage, and tests.
4. Map testplan items to sequences, checks, assertions, and coverage.
5. Identify filelist and simulation-flow updates.
6. List risks and open questions.

## Output

Provide UVM architecture, component responsibilities, test/sequence plan, scoreboard/coverage plan, and simulation update notes.
