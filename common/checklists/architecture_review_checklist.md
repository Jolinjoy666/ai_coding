# Architecture Review Checklist

- **Requirements mapping**: Each option maps to requirements.
- **Distinct options**: Options represent real implementation differences.
- **PPA tradeoffs**: Timing, area, power, latency, and throughput are compared.
- **Complexity tradeoffs**: RTL, verification, integration, and signoff complexity are compared.
- **Memory strategy**: RAM/register-file strategy is clear.
- **CDC/RDC strategy**: Clock/reset crossing risks are identified.
- **Recommendation justified**: Selected option has clear rationale.
- **Rejected options documented**: Rejected options have reasons.
