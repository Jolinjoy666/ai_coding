# 架构选项 A: 顺序执行架构

## 概述

最简单的架构，所有计算模块顺序执行，无流水线重叠。RISC-V 逐阶段配置触发，等待完成后再进入下一阶段。

## 数据通路

```
QKV投影 → FlashAttention → 残差+LayerNorm → MLP → 残差+LayerNorm
   ↓            ↓                ↓             ↓            ↓
 MAC阵列    MAC阵列+Softmax   加法器+LN硬件  MAC阵列+GELU  加法器+LN硬件
```

- 所有计算模块共享同一组 MAC 阵列
- 每个阶段独占硬件，完成后释放
- 无数据冒险，控制最简单

## 控制策略

- RISC-V 通过 APB 逐阶段配置和触发
- 每个阶段完成后产生中断
- RISC-V 在 WFI 状态等待，收到中断后配置下一阶段
- 固件控制流简单：循环 6 次（每层 6 个阶段）

## 存储架构

- Weight SRAM (16KB): 存储当前层权重
- Feature SRAM (16KB): 存储输入/中间/输出
- KV-Cache SRAM (8KB): 存储 K/V 缓存
- 三个 SRAM 独立 bank，无访存冲突

## 接口影响

- APB 单主设备，无仲裁需求
- UART 用于权重下载和结果回传
- GPIO 用于状态指示

## 性能估算

| 阶段 | 周期数 | 说明 |
|------|--------|------|
| QKV 投影 | 384 | 3×16×16×8 / 16 MAC |
| FlashAttention | 200 | 分块计算 + 在线 softmax |
| 残差+LN | 100 | 并行加法 + rsqrt |
| MLP (FC1+GELU+FC2) | 1024 | 矩阵乘 + 查表 |
| 残差+LN | 100 | 同上 |
| **单层合计** | ~1808 | |
| **2 层合计** | ~3616 | |
| **延迟 @100MHz** | ~36.2 μs | |

## PPA 权衡

| 指标 | 估算 |
|------|------|
| 面积 | ~176K gates |
| 功耗 | ~29 mW |
| MAC 利用率 | ~60%（顺序执行有空闲） |
| 吞吐量 | ~16.6K tokens/s |

## 验证影响

- 控制逻辑简单，验证复杂度低
- 无流水线冒险，覆盖率收敛快
- 固件调试简单

## 风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| MAC 利用率低 | 性能不达标 | 接受（入门版足够） |
| 吞吐量受限 | 不满足 P0 指标 | 升级到选项 B |

## 对应需求

- CR-001 到 CR-005（RISC-V 控制架构）
- CR-010 到 CR-018（Attention 引擎）
- CR-020 到 CR-022（MLP 引擎）
- CR-030 到 CR-033（LayerNorm 和残差）
- CR-070 到 CR-072（固件架构）
